LLM Safety Evaluation & Release Gating

The next evolution of the LLM Evaluation Framework — built for safety-critical deployment decisions, not just capability benchmarking. Where the first project measured reliability and consistency, this one asks a harder question: is this model behaviorally safe enough to release to real users?

Translates written safety policy into measurable evaluation categories, risk-weighted thresholds, and structured ship / conditional ship / block decisions — with an LLM judge that scores refusal quality and harmfulness alongside binary compliance. Every decision produces an auditable release report designed for Program Manager review.

Architecture
safety_policy.yaml → JSONL Datasets (62 prompts) → Model Completions → LLM Judge Evaluator → Release Gate → SHIP / CONDITIONAL / BLOCK
Scores: policy_compliance · refusal_quality (1–5) · harmfulness (1–5)
